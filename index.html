LawPatch

Language patches for legal instruments.

Why patches are useful

LawPatch is a collection of legal language modules that you can import into agreements. Rather than repeating all the legal formulas every time, you can patch a contract with "incorporation by reference".

Here's a simple example.

## Sensible boilerplate
The parties agree to the terms and conditions in the version of the text at
https://github.com/lawpatch/au-boilerplate/blob/d5f1348ff82b3e90b96d78a5ba1225367318c42e/sensible-boilerplate.md
only (not to any amendments).

Using patches saves time, reduces document length and results in more emphasis on terms unique to the transaction. When you need to review the patched language, it's just a click away.

All the patches below are open source (see license.md in each repo).

How to use patches

Check out the table of patches below. You can read the latest version of each patch by clicking "Read". You can display the Github permalink by clicking "Get".

When you click "Get" in the table below, the displayed link pinpoints a specific version of the text by including the repo name and the git commit hash. This is the intended approach to versioning.

Do not use a URL referring to the Github repo as a whole. This can easily be interpreted as importing the language as updated. We will be making substantive changes as we learn what works best, so there will be unintended consequences if you do this.

Jurisdiction Title Purpose Current
Australia boilerplate-aus Sensible end document boilerplate for Australian legal agreements. Read / Get
Australia interpretation-aus Sensible interpretation provisions for Australian legal agreements. Read / Get

Contributing patches

If you'd like to contribute a patch, we'd love to hear from you. Email Pat on pat@codepact.com with the language. You'll need to have the right intellectual property rights to contribute of course.

To get you thinking, we think this technique concept could work well with:

  • dispute resolution processes;
  • limitation of liability (particularly "no liability" positions); and
  • interpretation.

Be careful!

The patches are useful in many situations, but they are not applicable in all. That means you need to be careful with them. Before you use them in an agreement, speak with a lawyer - it should only take them 10 minutes to figure out if they're right for your situation (find a new lawyer if it takes any longer than that).

Canonical master branch

To make sure that links stay valid, we won't mess with commits to the master branch of a repo once they're made. To reduce the risk of mistake, we are using Github's protected branches feature.

We're putting some thought into how to host patch versions so that they'd outlast the vagaries of the tech industry (they rely on Github's existence right now).
